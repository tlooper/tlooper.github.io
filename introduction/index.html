欢迎来到SUP project! 这篇文章将带你了解以下问题：

[TOC]

SUP Project是什么？

​ 正如引言所说，我们设立SUP Project的目的是：将有用的信息进行整合，让人们能够轻易地获取所需的知识。遗憾的是由于开发人员目前仅仅是计算机本科生水平，SUP Project提供的知识目前仅限于计算机领域。并且基于互联网精神，我们的网站上所有的信息都是可以随意引用的。同时我们欢迎更多人完善我们的Project。

​ 声明：我们的SUP Project 并非一个公司实体，而更像是一个公益组织，SUP Project不对网站中任何文章负责，引用该文章也不代表本组织同意其中的观点、立场。

如何在SUP Project中查找自己需要的知识？

​ 我们将给优质的文章一个单独的前缀标识：SUP-number, 而其他的文章则会以SUP-extra 进行标识。并且对于每篇文章均有其对应的标签（tag），你同样可以通过标签来缩小查找范围。

如果对文章有疑问，如何反馈给作者？

​ 我们会在文章的末尾提供留言功能，并且给每个用户提供了一个单独的邮箱，你可以通过邮箱找到作者。

警告：我们不对邮箱的安全性作任何保证，所以请不要将该邮箱用作任何除文章内容探讨之外的私人内容。

我们除了查找文章，还能在SUP Project 中做什么？

​ SUP Project 还会设立社区板块，你可以在其中进行知识分享与讨论。因此我们会在每个人的账号中设置等级制度来区分不同人的权限。不同于其他的社区板块，只有为SUP Project 贡献优质内容的人才拥有独立开帖的权利。

关于等级制度：

  1. D级人员：注册账号后的基础等级。该等级的用户只能查看与检索内容，没有社区的使用权利。
  2. C级人员：注册账号后关联相应的个人网站后即可提升到C级人员的等级。可以在他人的帖子下进行评论。
  3. B级人员：如果您的文章被SUP Project 总站收录，即可晋升为B级人员，此时你就可以在社区进行发帖操作，并且可以管理你的帖子下的评论。
  4. A级人员：对社区贡献达到一定程度，（评判标准待定）将可以获得单独的称号，并且可以在社区开设自己的专题与用户群组。
  5. O5：O5成员即为SUP Project的管理员，每位O5成员都有自己专属的数字与字母的标识，并且将会在主站的成员页面显示。O5成员有审核文章、管理社区等一系列权利与义务。O5成员的变动与SUP Project的大型更改均由O5议会投票决定。

如何成为创作者？

成为SUP Project的创作者需要你有自己的个人网站。我们将会提供一套SUP Project的博客模板与详细的教程，让你能够独立建立自己的个人网站。在个人网站发布文章后，你可以在你的账户管理界面提交文章的URL，如果被SUP Project采用，我们就会将你的文章收录到我们的编号目录中，否则它将成为extra 文章，虽然可以在相应的tag中看到，但具有更低的优先级。

关于写一篇优质文章你需要知道的事情：

请善用标签对你的文章进行分类。目前开放的tag有：ICPC 、security、collection 、production、tutorial等。

对每个标签的解释：

ICPC：关于算法竞赛中用到的算法知识与题解。单独针对每一题的题解将会被定义为extra标签，请对算法进行归纳整理。

Security：安全竞赛方面的技巧与题解。原则同上，请在描述技巧时附上相应的题目与解析。

collection：对某一部分内容的搜集与整理。如果在文章中有其他链接请确保链接的有效性。

production：自己原创开发的产品，可以是某些插件或网站或别的什么项目。请确保原创性，否则请选择collection标签。

tutorial： 互联网教程，环境搭建、编程语言的使用等问题均属于该标签。如果仅仅是对他人的教程进行的归纳整理同样建议使用collection标签。

除了使用正确的tag以外，作者应该保证自己的文章有清晰的目录结构，并且文章内容具有时效性。需要注意被选为优质文章并非是一件永久的事情，如果你的文章失去时效性与准确性，我们将撤销它的编号。同时你在进行了更新之后我们也会恢复原先的编号或给予你新的编号。

在一篇文章成为了优质文章后，我们也会考察该文章的访问次数。如果优质文章的访问次数低于某一标准，我们将认定其内容“不够重要”，并降级成普通文章。同样，extra文章在访问量提升后也可以成为优质文章。
